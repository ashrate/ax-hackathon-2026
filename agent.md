# agent.md

이 저장소는 `AX 인재전쟁 2026` 해커톤 문제풀이를 위한 Codex 플러그인 제출물을 만드는 작업 공간입니다. Codex가 자동으로 읽는 에이전트 지침은 `AGENTS.md`에 있으며, 이 파일은 사람이 빠르게 확인하기 위한 요약입니다.

## 반드시 기억할 것

- 일반 앱이 아니라 해커톤 제출용 Codex 플러그인을 만든다.
- 세 개 회사를 정하고, 각 회사마다 별도 플러그인과 별도 `submission.zip`을 제출한다.
- 현재 트랙은 `channeltalk/`, `musinsa/`, `kakaopay-securities/`다.
- 각 zip에는 `src/`, 회사별 `README.md`, `logs/`가 들어가야 한다.
- `logs/`에는 AI와 나눈 전체 대화 로그 원본, 즉 모든 대화 로그를 포함한다.
- 로그는 편집, 발췌, 요약, 재작성하지 않는다.
- 예선 마감은 2026-07-10이다.
- 로그 파일을 원본 그대로가 아닌 사후 가공(편집·발췌·삭제)하여 제출하면 실격이다.
- 출처는 공개 자료로 AI가 확인할 수 있어야 하며, 비공개 정보·사적 경험·출처 없는 숫자는 근거로 인정되지 않는다.

## 5문항 기준

1. **무엇을, 누가, 어떤 상황에서 쓰나요?**
   플러그인 한 문장 요약과 기업의 누가, 어떤 상황에서, 어디서 막혀 쓰는지 쓴다. 제한은 800자다.

2. **왜 이 문제를 선택했나요?**
   문제 선택 이유와 실제 사용자가 어디서 왜 막히는지 쓴다. 제한은 800자이며 출처 URL이 필요하다.

3. **플러그인은 어떻게 작동하나요?**
   절차, 지식, 판단 기준, 정보가 부족하거나 잘 안 풀리는 상황의 동작을 쓴다. 제한은 1000자다. 세부 내용은 zip의 README에도 있어야 한다.

4. **AI를 어떻게 썼나요?**
   AI에게 맡긴 작업과 직접 판단한 부분을 구분한다. 막힌 지점, 해결 과정, 받아들이지 않은 AI 제안과 이유를 쓴다. 제한은 800자다.

5. **어떻게 검증했나요?**
   입력에서 결과로 이어지는 예시를 하나 이상 쓰고, 정상 상황과 예외 상황 검증, 의심한 점, 부족한 점, 테스트하며 고친 점을 쓴다. 제한은 800자다(제출 폼 기준 최종 확인).

## 작업 원칙

- 문제는 공개 자료로 입증한다.
- 출처 없는 숫자나 비공개 정보를 사실처럼 쓰지 않는다.
- README, `SKILL.md`, 5문항 답변은 같은 문제와 같은 작동 방식을 설명해야 한다.
- 제출 전 validator, 패키징, zip 내부 구조, 로그 포함 여부를 확인한다.
