# 무신사 제출용 플러그인

> AX 인재전쟁 2026 예선 제출 후보: MUSINSA Trend & Brand Scout

## 1. 무엇이 나오나요

무신사 MD/브랜드 스카우트가 카테고리나 트렌드 질문을 주면, 플러그인은 바로 회의용 산출물 폴더를 만듭니다.

```text
output/<카테고리>-<날짜>/
  scout-report.md
  signals-log.csv
  watchlist.md
  scorecards/
    <브랜드>.md
```

각 파일의 역할:

- `scout-report.md`: 후보 브랜드 5~10개 요약표와 이번 주 트렌드 신호 3~5개를 담은 주간 스카우트 리포트입니다.
- `scorecards/<브랜드>.md`: 브랜드별 1페이지 상세 카드입니다. 개요, 신호와 근거, 고객 반응, 반대 근거, 입점 적합성, 다음 액션을 적습니다.
- `signals-log.csv`: 날짜, 신호, 출처 URL, 출처 강도, 관련 브랜드, 메모를 누적하는 신호 원장입니다.
- `watchlist.md`: 신호가 부족하거나 다음 확인이 필요한 보류/모니터링 후보 목록입니다.

실제 예시는 `examples/여성-컨템포러리-2026-07-08/`에 들어 있습니다. 후보 8개 브랜드와 신호 24개를 공개 URL과 확인일 기준으로 정리했습니다.

## 2. 누가 언제 쓰나요

무신사 MD, 브랜드 스카우트, 카테고리/뷰티 담당자, 마케팅·전략 담당자가 주간 트렌드 리뷰나 신규 브랜드 검토를 준비할 때 씁니다. 공개 랭킹, 무신사/29CM 콘텐츠, 뉴스룸, 브랜드 공식 사이트, 업계 기사, 앱 리뷰, 공개 커뮤니티에 흩어진 신호를 사람이 바로 검토할 수 있는 스코어카드로 바꾸는 것이 목적입니다.

## 4. 작동 방식

1. 사용자가 준 카테고리와 날짜로 출력 폴더를 먼저 정의합니다.
2. 공개 자료에서 후보 브랜드와 트렌드 신호를 발굴합니다.
3. 각 신호에 URL, 확인일, 출처 강도, 접근 상태를 붙입니다.
4. 브랜드별 supporting evidence와 opposing evidence를 같이 기록합니다.
5. `즉시 검토`, `협업/콘텐츠 검토`, `모니터링`, `보류` 중 하나로 입점 관점 평가를 합니다.
6. 신호 부족 후보는 `watchlist.md`로 내리고, 출처 강도 3 이하 단독 근거는 `가설`로 표시합니다.
7. 최종적으로 `signals-log.csv`, `scorecards/*.md`, `scout-report.md`, `watchlist.md`를 생성합니다.

동적 페이지 처리:

- 무신사 랭킹, 아카이브, 추천, 상품 리스트처럼 동적 렌더링이 강한 페이지는 자동 추출 결과를 핵심 근거로 단정하지 않습니다.
- 대신 사용자가 화면으로 랭킹 위치, 리뷰, 좋아요, 가격대, 품절 여부를 확인할 입력 후보로 안내하고 `manual-check-required`를 남깁니다.

## 5. 폴백과 제한

- 무신사 내부 매출, 검색, CRM, 파트너 데이터 접근을 전제하지 않습니다.
- 공개 출처가 약하면 후보를 확정하지 않고 `watchlist.md`로 보냅니다.
- 로그인 전용 SNS나 비공개 자료는 핵심 근거에서 제외합니다.
- 브랜드 후보를 객관적 정답이나 입점 추천으로 단정하지 않습니다.
- 유행 예측은 확정 표현이 아니라 공개 근거가 있는 검토 가설로 씁니다.

## 6. 설치·실행 방법

```bash
python tools/make_submission.py musinsa
```

생성된 `dist/musinsa/submission.zip` 안의 `src/`가 플러그인 루트입니다. Codex에서 플러그인을 로드한 뒤 다음처럼 호출합니다.

```text
여성 컨템포러리에서 다음 주간 리뷰에 볼 브랜드 후보와 트렌드 신호를 공개 자료 기반 산출물 폴더로 만들어줘.
```

## 7. 검증 방법

제출 전 확인할 항목:

1. `src/.codex-plugin/plugin.json`이 validator를 통과한다.
2. `src/skills/main/SKILL.md`가 출력 계약을 종착점으로 삼는다.
3. 정상 입력 예시가 `examples/여성-컨템포러리-2026-07-08/`에 생성되어 있다.
4. `scout-report.md`, `signals-log.csv`, `watchlist.md`, `scorecards/*.md` 구조가 유지된다.
5. 신호 부족 후보와 출처 강도 3 이하 단독 근거가 `가설` 또는 `모니터링`으로 처리된다.
6. `logs/`에는 AI와 나눈 전체 대화 로그 원본이 포함되어 있다.
