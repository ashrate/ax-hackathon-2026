## 0. 대회 영상 자막 요약

- 출처: `docs/source-video.ko.vtt`
- 확인 날짜: 2026-07-08
- 화자/맥락: 채널코퍼레이션 CSO 이경훈으로 식별되는 화자가 채널코퍼레이션의 문제 정의, AX 인재상, 채널톡 트랙에서 보고 싶은 접근 방식을 설명한다.
- 회사가 정의한 본질 문제: 채널코퍼레이션은 "고객과 기업의 커뮤니케이션 문제"를 푸는 회사로 설명된다. 고객 불만과 고객 커뮤니케이션은 비즈니스 탄생부터 존재했고, 앞으로도 사라지지 않을 문제라는 맥락을 강조한다.
- AI/AX 관련 메시지: 특정 도구 이름보다 "본인이 가장 잘 쓸 수 있는 툴로 문제를 어떻게 풀었는가", "AI가 작업을 하고 사람이 판단한다", "왜 그런 판단을 했는가"를 본다고 설명한다. 표면적 자동화보다 문제의 본질을 탐구하는 몰입을 중요하게 본다.
- 채널톡 트랙 문제 힌트: "이커머스 고객사의 상담 문제"를 풀고 있고, 커머스가 가진 문제를 AI 에이전트로 해결하려 한다고 언급한다. 장바구니 이탈 고객에게 AI가 전화해 이유를 묻거나 쿠폰으로 구매를 유도하는 예시도 나온다.
- 인재/제출물 관점: 회사 규모는 작지만 "한 명이 열 명 할 수 있는 시대"라 인재 밀도를 높여야 한다고 한다. 해커톤 제출물은 단순 설명서보다, 문제를 파고들어 실제 워크플로우로 풀 수 있는지 보여주는 쪽이 적합하다.

## 5. 최종 선정 문제와 선정 이유

- 선정 날짜: 2026-07-08
- 선정한 문제: **ALF 도입 전 공개 사이트/FAQ/정책 문서를 구조화된 지식·규칙·테스트 세트로 바꾸는 수작업 문제**
- 선정 이유:
  - 대회 영상이 직접 "이커머스 고객사의 상담 문제"와 "AI 에이전트"를 언급한다.
  - 채널톡 공식 제품 설명과 도움말은 ALF 성공 조건을 `규칙`, `구조화된 지식`, `실행 가능한 태스크`, 배포 전 테스트로 제시한다. 이는 Codex가 문서를 읽고 워크플로우 산출물을 만드는 형태와 맞다.
  - 공식 AI 상담 시뮬레이션은 URL 입력과 웹 크롤링으로 상담 환경을 체험하게 한다. 즉, 공개 사이트 정보에서 AI 상담 지식 초안을 만드는 문제는 채널톡의 실제 제품 방향과도 맞다.
  - 공식 AI 고객 상담 사례 리포트 소개 페이지는 반복 문의와 프로모션 상담량 폭증을 CS팀 pain point로 제시한다. ZDNet 인터뷰의 배송지 변경, 주문 취소, 사이즈 문의 예시는 보조 근거로만 사용한다.
  - 1인 참가자가 2일 안에 만들 수 있다. 실제 Channel API, 고객사 계정, 비공개 데이터 없이도 Codex 스킬이 URL/문서 입력을 받아 지식 구조·규칙·테스트 질문·누락 리포트를 생성할 수 있다.
- 탈락시킨 후보와 탈락 이유:
  - 설치/연동 진단: 개발자에게 유용하지만 프레임워크 범위가 넓고, 대회 영상의 이커머스 상담/AI 에이전트 방향보다 구현 검증 범위가 커진다.
  - 알림 누락 점검: 실제 사용자 불만 근거는 있으나 OS/앱 설정을 직접 확인하기 어려워 체크리스트 수준에 머물 위험이 크다.
  - 알림톡 사전 검수: 문서 기반 정적 점검은 가능하지만 카카오 검수·채널 내부 설정 의존성이 있어 "Codex만으로 반복 실행"하기 애매하다.
  - 리뷰 VOC 티켓화: 공개 리뷰 수가 제한적이고, 고객사 업무 문제보다 채널톡 내부 제품 개선 문제에 가깝다.

## 6. 대상 사용자와 사용 상황

- 누가 쓰나: 이커머스 CS/CX 담당자, 쇼핑몰 운영자, 채널톡 AX 컨설턴트, ALF 도입을 준비하는 파트너/대행사.
- 어떤 상황에서 쓰나: 브랜드 사이트, FAQ, 배송·교환·환불 정책, 상품 안내 페이지는 있지만 이를 ALF 지식 베이스와 응답 규칙으로 어떻게 나눠야 할지 막힌 상황.
- 어디서·왜 막히나:
  - 홈페이지 정보는 사람이 읽기 위한 문서라 AI 상담용 폴더/규칙/이관 조건으로 구조화되어 있지 않다.
  - "배송 예정일", "환불 가능 여부", "사이즈 추천", "쿠폰/프로모션"처럼 잘못 답하면 고객 불만이나 정책 위반으로 이어지는 문의가 많다.
  - ALF가 실행 가능한 액션까지 수행하려면 단순 Q&A보다 트리거 조건, 제외 조건, 고객에게 다시 물어볼 정보, 상담원 이관 기준이 필요하다.
  - 운영자는 상담량이 폭증하는 시기에 지식 정리와 테스트 질문 설계를 동시에 해야 하므로, Codex가 초안을 생성하고 근거 URL/불확실성을 남기는 플러그인이 실용적이다.

## 7. 최종 추천 플러그인 개념

- 임시 이름: `channeltalk-alf-knowledge-planner`
- 한 문장 요약: 브랜드의 공개 고객지원 문서를 읽어 채널톡 ALF에 넣을 지식 구조, 응답 규칙, 액션 후보, 테스트 질문, 누락 문서를 생성하는 Codex 스킬 플러그인.
- 입력:
  - 브랜드 홈페이지 URL
  - FAQ/배송/교환/환불/멤버십/프로모션/상품 안내 URL 목록
  - 선택 입력: 운영자가 붙여넣은 정책 텍스트, 우선 자동화하고 싶은 문의 유형
- 출력:
  - `alf-knowledge-map.md`: 지식 폴더/문서 구조, 각 문서의 출처 URL, 우선순위
  - `alf-rules.md`: 기본 톤, 금지 답변, 정책별 규칙, 상담원 이관 기준
  - `alf-test-cases.md`: 고객 질문, 기대 답변 요지, 참조 URL, 실패 시 위험도
  - `alf-actions.md`: 주문 취소/배송조회/교환·반품 같은 액션 후보, 트리거/제외 조건, 필요한 외부 시스템/API
  - `gaps.md`: 사이트에 없거나 모호한 정보, 운영자 확인 질문
- 실패/정보 부족 시 동작:
  - 접근 불가 URL은 건너뛰고 실패 URL과 이유를 기록한다.
  - 출처가 없는 정책은 "미확인"으로 표시하고 ALF 규칙에 넣지 않는다.
  - 결제/환불/개인정보처럼 위험도가 높은 주제는 확정 답변이 아니라 상담원 이관 또는 추가 확인 질문으로 제안한다.
  - 실제 주문 취소 등 액션은 API 키 없이 구현하지 않고, 액션 설계 초안과 필요한 API/권한만 적는다.

(작성 날짜: 2026-07-08)

`채널톡 ALF 지식·규칙 설계 플러그인`은 이커머스 CS/CX 담당자나 채널톡 AX 컨설턴트가 브랜드 홈페이지, FAQ, 배송·교환·환불 정책 URL을 Codex에 넣으면 ALF에 넣을 지식 폴더, 응답 규칙, 상담원 이관 기준, 액션 후보, 테스트 질문을 생성하는 스킬입니다. 프로모션·신상품·배송 이슈로 반복 문의가 늘었는데 개발자 없이 AI 상담 도입을 준비해야 할 때 씁니다. 사용자는 흩어진 공개 페이지를 읽어 AI가 안전하게 답할 수 있는 구조화 지식과 금지 규칙으로 바꾸는 데 막힙니다. 이 플러그인은 출처 URL, 불확실한 항목, 운영자 확인 질문을 함께 남겨 바로 검토 가능한 ALF 설정 초안을 만듭니다.

(작성 날짜: 2026-07-08)

채널톡 트랙 영상은 회사가 "이커머스 고객사의 상담 문제"를 AI 에이전트로 풀고 있다고 말합니다. 공식 제품 페이지는 ALF가 제대로 답하려면 올바른 규칙, 구조화된 지식, 실행 가능한 태스크가 필요하다고 설명합니다(https://channel.io/kr/talk). 공식 도움말도 지식은 ALF가 참조할 문서·파일·웹사이트 자료이고, 규칙은 상황별 지시문이라고 안내합니다(https://docs.channel.io/help/ko/articles/%EC%A7%80%EC%8B%9D-803f6ac9, https://docs.channel.io/help/ko/articles/%EA%B7%9C%EC%B9%99-b43e19a1). 채널톡은 URL 입력으로 홈페이지 자료를 수집해 AI 상담을 사전 체험하는 기능도 출시했습니다(https://channel.io/kr/blog/articles/crawling-press-9b611f25). 반복 문의와 프로모션 상담량 폭증을 겪는 CS팀을 공식 리포트 독자로 제시한 점도 이 문제를 뒷받침합니다(https://channel.io/kr/blog/articles/ai-cs-best-cases-6a7fd39d). 따라서 공개 사이트 정보를 ALF용 지식·규칙·테스트로 바꾸는 일은 실제 제품 방향과 고객 pain point가 만나는 지점이고, API 키 없이 Codex 스킬로 검증 가능한 문제라 선택했습니다.
