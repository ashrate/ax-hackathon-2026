# Apply Guide - 바잇미

## 사전 준비

- 이 폴더의 모든 파일을 검토한다.
- `gaps.md`의 미확인 정책과 문서 충돌을 운영자가 먼저 확정한다.
- 상품별 정책이 자주 바뀌면 대표 상품뿐 아니라 고문의 상품군을 추가로 수집한다.

## 1. 지식 문서 등록

1. 채널톡 관리자에서 ALF가 참조할 지식 또는 도큐먼트 관리 화면을 연다.
2. `alf-knowledge/` 아래 폴더 구조를 그대로 만든다.
3. 각 `.md` 파일을 같은 카테고리 문서로 등록한다.
4. 문서 하단의 출처 URL과 확인일을 삭제하지 않는다.
5. 상품별 예외 문서는 공통 정책보다 우선 참고하도록 문서 제목에 상품명을 유지한다.

## 2. 규칙 등록

1. `alf-rules.md`의 기본 톤과 공통 금지 답변을 먼저 등록한다.
2. 정책별 규칙 블록을 배송, 주문취소, 교환/반품/환불, 쿠폰/마일리지 순으로 등록한다.
3. HAX 제주/도서산간 추가비 충돌처럼 `gaps.md`에 있는 항목은 단정 답변 금지 규칙으로 유지한다.

## 3. 상담원 이관 기준 등록

1. `alf-escalation.md`의 즉시 이관 트리거를 상담원 연결 조건으로 옮긴다.
2. 오배송, 누락, 불량, 환불 지연, 상품준비중 이후 주문 변경, 개인정보 문의는 고위험 트리거로 둔다.
3. 이관 문구는 고객에게 필요한 준비물만 안내하고 처리 결과를 약속하지 않는다.

## 4. 테스트 케이스 실행

1. `alf-test-cases.csv`를 스프레드시트로 열어 30개 이상 질문이 있는지 확인한다.
2. `오답위험도=높음` 질문을 먼저 테스트한다.
3. 기대답변요지와 실제 ALF 답변을 비교한다.
4. 출처 없는 답변, 상품별 정책 단정, 주문 상태 추정이 나오면 해당 지식 또는 규칙을 수정한다.

## 5. 운영 전 확인

- 모든 확정 답변이 공개 출처 또는 운영자 확인 정책에 근거하는지 확인한다.
- `gaps.md`의 미확인 항목이 실제 운영에서 허용 가능한지 확인한다.
- 고객에게 비용이 발생하는 배송비, 반품비, 환불 지연 안내는 상담팀 최종 검토를 받는다.

## 6. 업데이트 주기

- 이벤트 쿠폰과 프로모션은 변경 가능성이 높으므로 운영자가 캠페인 시작 전 확인한다.
- 명절 배송 휴무와 물류 지연 공지는 공지 발생 시 즉시 업데이트한다.
- 상품별 배송비와 반품 가능 기간은 신상품 등록 또는 공급사 정책 변경 시 다시 수집한다.
