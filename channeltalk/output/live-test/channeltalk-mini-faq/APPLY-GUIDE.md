# Apply Guide - live-test-mini-faq

## 사전 준비

- 이 폴더는 사용자 제공 공개 FAQ 텍스트만 근거로 만든 미니 테스트 산출물이다.
- 운영 전 `gaps.md`의 원문 FAQ URL과 미확인 정책을 확인한다.
- 실제 채널톡 계정 설정을 자동 변경하지 않는다.

## 1. 지식 문서 등록

- `alf-knowledge/` 아래 문서를 카테고리별로 확인한다.
- 배송, 교환/반품, 쿠폰 문서를 ALF 지식 문서로 옮긴다.
- 각 문서의 출처가 `사용자 제공 공개 FAQ 텍스트`임을 확인하고, 원문 URL이 확보되면 출처를 갱신한다.

## 2. 규칙 등록

- `alf-rules.md`의 기본 톤, 공통 금지 답변, 정책별 규칙을 ALF 규칙 입력창에 맞게 복사한다.
- 공개 FAQ에 없는 항목은 답변하지 않도록 금지 답변을 유지한다.

## 3. 상담원 이관 기준 등록

- `alf-escalation.md`의 즉시 이관 트리거와 이관 문구를 운영 정책에 맞게 검토한다.
- 주문별 배송 상태, 상품 상태 판단, 쿠폰 복구 여부는 상담원 확인으로 둔다.

## 4. 테스트 케이스 실행

- `alf-test-cases.csv`를 열어 정상 질문, 예외 질문, 이관 질문을 차례로 테스트한다.
- 오답위험도 `높음` 질문은 ALF가 단정하지 않고 확인 또는 이관으로 답하는지 확인한다.

## 5. 운영 전 확인

- 원문 FAQ URL을 확보한다.
- 배송비, 환불 처리 기간, 교환/반품 비용, 쿠폰 예외 정책을 운영자가 확인한다.
- 실제 고객에게 노출하기 전에 상담원 이관 문구와 운영 시간을 검수한다.

## 6. 업데이트 주기

- FAQ나 정책이 바뀌면 관련 지식 문서, 규칙, 테스트 케이스, gaps를 함께 갱신한다.
- 쿠폰 이벤트나 프로모션 기간에는 쿠폰 복구 정책을 별도로 재확인한다.
