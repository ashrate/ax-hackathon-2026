# ChannelTalk ALF Knowledge Planner

## 목적

채널톡의 공식 영상 힌트인 “이커머스 고객사의 상담 문제”와 “AI 에이전트”에 맞춰, 공개 사이트/FAQ/정책 문서를 ALF 도입 전 바로 적용 가능한 산출물 폴더로 변환한다. 목표는 감사 문서가 아니라, CS 담당자가 채널톡 관리자 화면에 옮겨 넣고 테스트할 수 있는 프로토타입 결과물이다.

모든 실행의 종착점은 `output/<브랜드명>/` 출력 계약이다. 계약 세부는 저장소의 `docs/output-spec.md`를 따른다.

## 먼저 읽을 자료

작업을 시작할 때 `references/evidence.md`를 기준 근거 맵으로 삼고, 출력 스키마는 `docs/output-spec.md`를 따른다. 사용자가 새 공개 출처를 주면 출처 강도와 확인 필요 여부를 표시한다.

## 발동 조건

다음 요청에서 사용한다.

- 이커머스 FAQ, 배송/교환/환불 정책, 상품 안내, 쿠폰/프로모션 문서를 ALF용 산출물 폴더로 바꿀 때
- 채널톡 트랙의 5문항 답변, README, 검증 기록을 이 문제 기준으로 정리할 때
- 공개 출처가 충분한지, 답변 규칙과 상담원 이관 기준이 안전한지 점검할 때

## 입력

- 브랜드명
- 공개 URL 또는 사용자가 붙여넣은 FAQ/정책/상품 설명
- 선택 입력: 대상 범위(배송, 교환/환불, 사이즈, 쿠폰, 주문 변경, 상담원 이관 등)
- 선택 입력: 금지 답변, 브랜드 톤, 상담원 확인이 필요한 정책

## 수행 단계

### 1. 입력 수집

1. 브랜드명과 URL 목록을 확인한다.
2. URL을 FAQ, 배송, 교환/반품/환불, 주문/회원, 쿠폰/적립금, 상품별 정책, 기타로 분류한다.
3. 개인정보, 주문번호, API 키, 관리자 화면 캡처가 있으면 사용하지 말고 공개 정책만 다시 요청한다.

### 2. 공개 문서 수집·분류

1. 각 URL의 접근 가능 여부와 확인일을 기록한다.
2. 로그인 없이 읽을 수 있는 본문, 공개 AJAX 응답, 정적 문서만 사용한다.
3. 문서를 `배송`, `교환`, `반품`, `환불`, `주문취소`, `주문변경`, `회원/계정`, `쿠폰/적립금`, `상담원 이관` 같은 상담 의도 단위로 나눈다.
4. 상품별 정책이 다르면 공통 정책으로 합치지 않고 상품별 예외로 남긴다.
5. 접근 실패, 부분 수집, 미확인, 충돌은 `gaps.md` 후보로 기록한다.

### 3. 계약된 출력 생성

반드시 아래 구조를 만든다.

```text
output/<브랜드명>/
  alf-knowledge/
    <지식 카테고리>/
      <문서명>.md
  alf-rules.md
  alf-test-cases.csv
  alf-escalation.md
  gaps.md
  APPLY-GUIDE.md
```

생성 규칙:

1. `alf-knowledge/`에는 카테고리별 문서를 만들고 각 문서 하단에 출처 URL과 확인일을 쓴다.
2. `alf-rules.md`에는 기본 톤, 공통 금지 답변, 정책별 규칙, 고위험 주제 처리 규칙을 채널톡 규칙 입력창에 복사 가능한 블록으로 쓴다.
3. `alf-test-cases.csv`에는 헤더 `질문,의도분류,기대답변요지,근거URL,오답위험도`와 테스트 질문 30개 이상을 만든다.
4. `alf-escalation.md`에는 즉시 이관 트리거, 추가 질문 후 이관, 이관 제외 조건, 이관 문구를 쓴다.
5. `gaps.md`에는 사이트에 없거나 모호한 정책과 접근 실패 URL을 운영자 확인 질문으로 정리한다.
6. `APPLY-GUIDE.md`에는 산출물을 채널톡 관리자 화면에 적용하고 테스트하는 순서를 쓴다.

### 4. 자체 품질 점검

완료 전 반드시 확인한다.

1. 확정 정책 문장에 출처 URL/확인일이 있거나 `미확인` 표시로 분리되어 있는지 점검.
2. 각 `alf-knowledge/**/*.md` 문서에 출처 URL과 확인일 존재.
3. `alf-test-cases.csv`가 CSV 파서로 읽히고 30행 이상 존재.
4. `alf-test-cases.csv`의 오답위험도가 `낮음`, `중간`, `높음` 중 하나.
5. 고위험 주제는 단정 답변이 아니라 조건 확인 또는 상담원 이관으로 처리.
6. `gaps.md`의 미확인 항목이 `alf-rules.md`의 금지 답변 또는 이관 규칙과 연결됨.

## 출력 형식

기본 출력은 `output/<브랜드명>/` 폴더 자체다. 대화로 보고할 때는 출력 계약 요약, 생성된 트리, 파일 수, 테스트 질문 수, 품질 점검 결과만 짧게 보고한다.

## 주의사항

- 채널톡 내부 ALF 설정이나 고객사의 비공개 주문/CRM 데이터를 안다고 쓰지 않는다.
- 환불, 취소, 배송 예정일, 쿠폰 적용처럼 고객 불만으로 이어질 수 있는 항목은 공식 문서 근거 없이는 단정하지 않는다.
- 공개 출처가 약한 주장은 `미확인` 또는 `운영자 확인 필요`로 표시한다.
- API 키, 계정 정보, 개인정보는 대화와 로그에 남기지 않는다.
- 로그 제출 조건 때문에 대화 내용을 임의로 편집하거나 요약본만 제출하라고 안내하지 않는다.
