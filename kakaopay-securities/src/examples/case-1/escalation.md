# 이관 판단

## 라우팅 결과

| 구분 | 판단 |
| --- | --- |
| 기본 라우팅 | 1차 상담사 처리 가능 |
| 이관 필요 여부 | 현재 문의만으로는 이관 불필요 |
| 이관 사유 | 고객이 앱 장애, 주문 불가, 보상 요구, 실제 손실 확정을 주장하면 주문장애 기준 확인으로 이관 |
| 고객 안내 톤 | 체결 구조 설명 + 최신 공지 확인 + 손실 단정 금지 |

## 이관 조건

| 조건 | 이관처 | 근거 |
| --- | --- | --- |
| 고객이 "아예 주문할 수 없었다"거나 "앱/ARS/고객센터 모두 불가했다"고 주장 | 주문장애·보상 접수 담당 또는 고객센터 | 주문장애는 회사 주문 시스템 장애로 MTS와 비상주문 등 어떤 방법으로도 주문이 불가능한 경우로 제한됨[^1] |
| 고객이 보상을 요구 | 주문장애 기준 확인 절차 | 비상주문 시도, 손실 확정, 신청기한 등 조건 확인 필요[^1] |
| 고객이 특정 종목을 계속 보유하거나 팔아야 하는지 질문 | 투자자문 라우팅 안내 | 1:1 투자자문은 정식 등록 투자자문업자 영역이라는 금융위원회 안내가 있음[^2] |
| FAQ와 현재 앱 표시가 다르다고 주장 | 공식 확인 필요 | 해외주식 소수점 거래 설명서와 유의사항은 공지로 개정될 수 있음[^3] |

## 상담사 처리 문구

현재 문의는 소수점매매 체결 방식 안내로 1차 처리합니다. 고객이 장애나 보상 가능성을 주장하면 주문장애 기준 충족 여부를 확인해야 하므로 발생 시각, 접속 채널, 오류 화면, 비상주문 시도 여부를 인증된 고객센터 채널에서 확인하도록 안내합니다.
