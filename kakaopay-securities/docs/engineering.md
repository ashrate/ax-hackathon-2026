# 카카오페이증권 — 엔지니어링 기록

> 이 파일은 submission.zip에는 포함되지 않는 작업 기록이며, 문항 ③·④와 README.md 작성의 원천 자료다.

## 1. 플러그인 설계 개요

- 플러그인명: KakaoPay Securities Investor Reassurance Copilot
- 해결 문제: 초보 투자자가 “사야 하나/팔아야 하나”를 물을 때 직접 투자 추천 없이 불안을 분해하고, 공식 근거로 다음 확인 행동을 안내하기 어렵다.
- 스킬 구성: `src/skills/main/SKILL.md` 하나로 실행한다. 근거 맵은 `src/references/evidence.md`에 둔다.
- 핵심 입력: 투자자 질문, 상담 초안, 매수·매도 불안 상황, 선택적으로 소수점 거래/주문/공지/FAQ 맥락.
- 핵심 출력: 질문 유형 분류, 투자 추천 금지 판단, 명확화 질문, 공식 근거 체크리스트, 고객 답변 초안, 금지 표현 QA, 이관 기준.

## 2. 작동 절차

1. 입력 질문을 투자 추천 요청, 거래 구조 이해, 손실 불안, 상담 품질 점검으로 분류한다.
2. 특정 종목, 가격, 타이밍, 매수·매도 지시를 요구하면 추천을 거절하고 체크리스트형 사고 과정으로 전환한다.
3. 투자 목적, 기간, 손실 감내도, 보유 여부, 이미 확인한 근거, 주문 방식 이해 여부를 묻는다.
4. 공식 FAQ, 공지, 금융당국 자료에서 확인 가능한 사실과 확인 불가능한 판단을 분리한다.
5. 고객용 답변 초안을 만들 때 수익 보장, 목표가, 매수/매도 지시, 보상 단정 표현을 검사한다.
6. 공식 근거가 부족하거나 최신성이 의심되면 상담사 또는 공식 고객센터 확인으로 이관한다.
7. 5문항 또는 README 작성 요청이 있으면 같은 문제 정의와 작동 방식이 유지되도록 초안을 낸다.

## 3. 판단 기준

- 공식 영상 자막의 “초보 투자자의 매수·매도 불안”과 “납득하고 안심할 과정 설계” 힌트를 최우선으로 둔다.
- 공식 FAQ, 공지, 금융위원회·금융감독원 등 공개 규제 자료를 우선한다.
- 특정 종목·가격·타이밍 판단은 하지 않는다.
- 감정적 안심과 사실 설명을 분리한다.
- 최신 공지 확인이 필요한 항목은 답변하지 않고 재확인으로 넘긴다.

## 4. 정보 부족·실패 상황 폴백

- 사용자가 추천을 요구할 때: 추천을 거절하고 목적·기간·위험 질문으로 전환한다.
- 공식 출처가 부족할 때: “공식 확인 필요”로 표시하고 상담 이관 문구를 만든다.
- 계좌·보유 종목 등 비공개 정보가 필요할 때: 데이터를 요구하지 않고 일반적 체크리스트만 제공한다.
- 상담 초안이 법적 단정처럼 보일 때: 금지 표현을 표시하고 완화된 문장으로 바꾼다.
- 공지 최신성이 의심될 때: 확인일과 재확인 URL을 출력한다.

## 6. AI 활용 기록

- AI에 맡긴 작업: 자막·FAQ·공지·금융당국 자료 요약, 후보 문제 비교, 금융 규제 리스크가 낮은 출력 구조 설계, 금지 표현 목록화.
- 직접 판단한 부분: 소수점 거래 상담 코파일럿만으로는 영상의 매수·매도 불안을 정면으로 다루지 못하므로, 비추천형 투자 불안 분해 플러그인으로 바꿨다.
- 받아들이지 않은 AI 제안: 종목 분석이나 매매 판단 보조 기능은 사용자가 원할 수 있지만 투자권유 오해가 커서 제외했다.
- 이번 감사에서 AI에 맡긴 작업: 웹 검색, 공식 HTML·FAQ 원문 접근 확인, 보조 근거와 공식 근거의 가중치 재분류, 반증 후보 탐색.
- 이번 감사에서 직접 판단한 부분: 문항② 최종 URL은 PDF와 앱 리뷰가 아니라 로그인 없이 HTML 본문이 확인된 카카오/금융위/KDI 출처로 제한했다.
- 이번 감사에서 받아들이지 않은 제안: 종목추천서비스 PDF를 핵심 URL로 쓰는 방안은 PDF 텍스트 접근성이 낮고 플러그인 의도와 중복 오해가 커서 반증 검토 항목으로만 남겼다.

## 7. 답변 초안

### 문항 ③ 플러그인은 어떻게 작동하나요? (1000자 제한)

이 플러그인은 초보 투자자의 “사야 하나/팔아야 하나”, “손해 본 것 같아 불안하다”, “주문 방식이 이해되지 않는다”는 질문을 투자 추천 없이 분해합니다. 먼저 질문을 투자 추천 요청, 거래 구조 이해, 손실 불안, 상담 품질 점검으로 분류합니다. 특정 종목, 가격, 타이밍, 매수·매도 지시를 요구하면 추천을 거절하고 목적, 기간, 손실 감내도, 보유 여부, 이미 확인한 근거, 주문 방식 이해 여부를 묻는 체크리스트로 전환합니다. 답변은 카카오페이증권 FAQ·공지, 카카오페이 고객센터, 금융당국 자료처럼 공개 공식 근거로 확인 가능한 사실과 확인 불가능한 판단을 분리합니다. 고객 답변 초안에는 수익 보장, 목표가, 매수/매도 지시, 보상 단정 표현이 없는지 검사합니다. 근거가 부족하거나 최신성이 의심되면 답을 단정하지 않고 공식 고객센터 확인 또는 상담사 이관으로 넘깁니다.

### 문항 ④ AI를 어떻게 활용했나요? (800자 제한)

AI에는 유튜브 자막에서 출제자의 문제 의도를 뽑고, 카카오페이증권 FAQ·공지·금융당국 자료를 근거별로 정리하게 했습니다. 후보 문제로 소수점 거래 상담, 투자 불안 분해, 상담 답변 QA를 비교했고, 각 후보의 규제 리스크와 구현 가능성도 검토하게 했습니다. 직접 판단한 부분은 최종 문제 선택입니다. 소수점 거래 상담은 공식 근거가 강하지만 영상의 “매수·매도 불안”을 우회하는 느낌이 있어, 투자 추천 없이 불안을 분해하는 코파일럿으로 바꿨습니다. 받아들이지 않은 제안은 종목 분석이나 매매 타이밍 보조입니다. 사용자는 원할 수 있지만 투자권유 오해가 커서 특정 종목·가격·시점 판단을 금지하고 공식 근거와 체크리스트 중심으로 범위를 줄였습니다.
