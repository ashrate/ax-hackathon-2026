# 확인 체크리스트

## 케이스 요약

| 항목 | 내용 |
| --- | --- |
| 케이스 | case-2 |
| 고객 문의 | "아침에 앱이 안 열려서 주문을 못 했어요. 보상해 주나요?" |
| 문의유형 | 앱 접속 장애·보상 |
| 핵심 불안 | 주문 기회 상실과 보상 가능성 |
| 우선 처리 | 보상 단정 금지, 주문장애 기준 확인, 증빙·비상주문 여부 확인 |

## 고객에게 물어볼 확인 질문

| 질문 | 이유 | 필수 여부 |
| --- | --- | --- |
| 앱 접속이 안 된 날짜와 대략적인 시간대가 언제인가요? | 공지된 지연 또는 장애 시간과 대조 필요 | 필수 |
| 앱 외에 고객센터 또는 ARS 비상주문을 시도하셨나요? | 주문장애 기준과 보상 제외 여부 확인 | 필수 |
| 오류 화면 캡처나 동영상이 있나요? | 공식 접수와 사실관계 확인 자료 | 필수 |
| 매수 주문이었나요, 매도 주문이었나요? | 보상 기준은 손실 확정 방식과 주문 유형에 영향을 받음 | 필수 |
| 주문하려던 가격과 수량은 인증된 고객센터 채널에서 확인 가능하신가요? | 공개 채널에 개인정보·주문정보를 남기지 않기 위함 | 필수 |
| 해당 시간대에 카카오페이증권 공지가 있었는지 확인했나요? | 공지된 서비스 지연 여부 확인 | 권장 |
| 주문 실패로 인한 투자 판단 변경이 필요한가요? | 투자자문 요구로 전환되는지 확인 | 권장 |

## 다음 행동 단계

1. 공개 채널에는 개인정보와 주문번호 전체값을 받지 않는다.
2. 고객에게 발생 시각, 접속 채널, 오류 화면, 비상주문 시도 여부를 인증된 고객센터 채널로 제출하도록 안내한다.
3. 카카오페이증권 주문장애 기준에서 장애 정의와 보상 제외 기준을 확인한다.
4. 해당 날짜의 공지사항 또는 장애 안내가 있는지 확인한다.
5. 보상 가능 여부를 답변하지 말고 "공식 접수 후 기준 충족 여부 확인"으로 유지한다.
6. 투자 판단 질문이 이어지면 추천 없이 목적, 기간, 손실 감내도 확인으로 전환한다.
