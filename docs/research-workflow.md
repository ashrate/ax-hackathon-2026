# AX 인재전쟁 리서치 워크플로우

이 문서는 세 회사 트랙을 같은 방식으로 검증하기 위한 공통 리서치 프로토콜이다. 현재 보유한 1차 자료는 회사별 유튜브 자막(`docs/source-video.ko.vtt`)이며, 이 자막은 출제자가 문제 의도를 직접 설명한 자료로 최우선 가중치를 둔다.

## 1. 자료 가중치

| 자료 유형 | 가중치 | 사용 목적 |
| --- | --- | --- |
| 공식 유튜브 자막 | 5 | 출제자의 문제 의도, 금지해야 할 방향, 평가 관점을 잡는다. |
| 회사 공식 사이트, 뉴스룸, 제품 문서, 채용공고 | 4 | 회사가 공개적으로 말한 제품 방향과 실제 업무 맥락을 입증한다. |
| 고객센터, FAQ, 공지, 약관, 개발자 문서 | 4 | 플러그인이 따라야 할 운영 규칙과 공식 판단 기준을 확보한다. |
| 앱 리뷰, 공개 커뮤니티, 기사, 산업 자료 | 3 | 직원·고객·산업 사용자가 어디서 막히는지 보조 근거로 쓴다. |
| 검색 결과, 블로그, 2차 요약 | 2 | 후보 탐색에만 쓰고, 핵심 주장의 단독 근거로 쓰지 않는다. |
| AI 요약, NotebookLM 답변 | 1 | 구조화와 누락 확인용으로만 쓴다. 출처 원문으로 역추적되지 않으면 폐기한다. |

## 2. 기본 절차

1. `docs/source-video.ko.vtt`에서 출제자가 직접 말한 문제 힌트, 평가 관점, 피해야 할 풀이를 뽑는다.
2. 회사 공식 자료에서 영상 힌트와 같은 방향의 공개 근거를 찾는다.
3. 고객·직원·산업 사용자가 실제로 막히는 장면을 공개 자료로 보강한다.
4. 후보 문제를 3개 이상 만들고, 영상 정합성, 공개 근거 강도, 2일 내 구현 가능성, 플러그인 반복 실행성으로 점수화한다.
5. 최종 문제를 하나 고른 뒤 `docs/research.md`, `docs/engineering.md`, `docs/verification.md`, `src/references/evidence.md`, `README.md`, `src/skills/main/SKILL.md`가 같은 문제를 설명하는지 맞춘다.
6. 제출 전에는 validator, 패키징, zip 내부 구조, 로그 포함 여부, 예시 입력 결과를 확인한다.

## 3. insane-research 사용 원칙

insane-research류 도구는 단순 검색이 아니라, 접근이 어렵거나 흩어진 공개 자료를 수집·비교할 때 쓴다. 예시는 JS 렌더링 페이지, 공개 리뷰, 뉴스룸 아카이브, 검색 결과 묶음, 커뮤니티 페이지다.

반드시 남길 것:

- 접근한 URL
- 확인일
- 수집 방법
- 해당 자료가 입증하는 주장
- 원문에서 직접 확인한 문장이나 표의 위치
- 불확실하거나 접근이 실패한 자료

금지할 것:

- 로그인·권한이 필요한 비공개 자료를 근거처럼 쓰기
- AI가 요약한 내용만 보고 출처 원문 확인 없이 사실로 쓰기
- 출처 없는 수치, 내부 사정 추정, 경쟁사 비교 단정

## 4. NotebookLM 사용 원칙

NotebookLM은 자막, 공식 문서, FAQ, 기사 PDF/웹페이지를 넣고 빠르게 질의응답하며 누락을 찾는 보조 도구로 쓴다. 단, NotebookLM의 답변은 제출 근거가 아니다.

사용 가능한 작업:

- 유튜브 자막에서 출제자 의도 요약
- 회사별 후보 문제 비교
- 5문항 초안의 빠진 논리 확인
- 자료 간 모순 후보 찾기

반드시 할 일:

- NotebookLM 답변을 `docs/research.md`의 실제 URL 또는 `source-video.ko.vtt` 내용으로 역추적한다.
- 역추적이 안 되는 문장은 제출 답변에서 제거한다.
- 출처 확인일과 URL을 별도로 기록한다.

## 5. 회사별 현재 리서치 방향

| 트랙 | 최우선 영상 힌트 | 현재 최종 문제 가설 | 다음 리서치 초점 |
| --- | --- | --- | --- |
| 채널톡 | 이커머스 고객사의 상담 문제와 AI 에이전트 | 공개 FAQ/정책/상품 정보를 ALF용 지식·규칙·테스트로 구조화하는 수작업 문제 | ALF, AI 상담, 웹 크롤링, 이커머스 반복 문의 공식 근거 강화 |
| 무신사 | 고객이 좋아할 브랜드와 놓치지 말아야 할 트렌드 발굴 | MD/브랜드 스카우트가 공개 신호에서 신규 브랜드·트렌드 후보를 반복 발굴하기 어려운 문제 | 뉴스룸, 랭킹, 리뷰, 공개 커뮤니티 신호의 출처 강도와 스코어 기준 정리 |
| 카카오페이증권 | 초보 투자자의 매수·매도 불안, 상담 지원 AI | 투자 추천 없이 불안을 분해하고 공식 근거로 다음 확인 행동을 만드는 문제 | 금융 소비자 보호, FAQ/공지, 투자권유 금지 표현, 상담 QA 기준 강화 |

## 6. 폴더별 역할

각 회사 폴더는 다음 계약을 따른다.

- `docs/research.md`: 문제 선정 근거, 후보 비교, 문항 1·2의 원천 자료
- `docs/engineering.md`: 플러그인 작동 방식, 폴백, AI 활용 기록, 문항 3·4의 원천 자료
- `docs/verification.md`: 예시 입력, 정상·예외 검증, 한계, 문항 5의 원천 자료
- `src/references/evidence.md`: 제출 zip 안에서 플러그인이 참고할 근거 맵
- `src/skills/main/SKILL.md`: 실제 Codex 플러그인 실행 절차
- `README.md`: 제출자가 심사자에게 설명할 문제, 작동 방식, 검증 요약
- `logs/`: AI 대화 로그 원본. 편집·발췌·삭제하지 않는다.
